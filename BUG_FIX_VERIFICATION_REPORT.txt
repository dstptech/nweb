FILE: dstp_backend/apps/authentication/views.py (lines 205-223)

PROBLEM:
  ❌ Service returns dict: {'access': '...', 'refresh': '...'}
  ❌ View expected string: new_access_token
  ❌ Client receives invalid response format

SOLUTION APPLIED:
  ✅ Changed variable: new_access_token → token_data
  ✅ Extract both tokens: token_data['access'] and token_data.get('refresh')
  ✅ Send both tokens to client
  ✅ Updated docstring to show correct response format

CODE BEFORE:
  try:
      new_access_token = AuthService.refresh_access_token(refresh_token)  # Gets dict
  except AuthenticationFailedError as e:
      ...
  return Response({
      'success': True,
      'data': {'access': new_access_token}  # ❌ Tries to use dict as string
  }, status=status.HTTP_200_OK)

CODE AFTER:
  try:
      token_data = AuthService.refresh_access_token(refresh_token)  # Gets dict
  except AuthenticationFailedError as e:
      ...
  return Response({
      'success': True,
      'data': {
          'access': token_data['access'],  # ✅ Extracts from dict
          'refresh': token_data.get('refresh', refresh_token),  # ✅ New refresh token
      }
  }, status=status.HTTP_200_OK)

VERIFICATION:
  ✅ Fixed variable typing
  ✅ Correct dict unpacking
  ✅ Both tokens sent to client
  ✅ Client can update its stored refresh token
